#  Use-case и activity диаграммы

Этот раздел описывает взаимодействие пользователей с системой и логику основных рабочих процессов.



## 1. Use-case диаграмма

Диаграмма вариантов использования (Use-case diagram) иллюстрирует основные действия, которые исследователь (Researcher) может выполнять с помощью библиотеки.


```{image} ./_static/Use-case-researcher.png
:alt: Select Parameters
:width: 500px
:align: center
```



- **Актор:**
  - **Researcher (Исследователь):** Основной пользователь системы. Это может быть аналитик данных, ученый или инженер, которому необходимо проанализировать данные для выявления лежащих в их основе скрытых групп (компонент).
- **Варианты использования (Use Cases):**
  1. **Estimate number of components (Оценить количество компонент):**
     - **Описание:** Исследователь использует библиотеку для автоматической оценки оптимального числа компонент в смеси на основе предоставленной выборки данных. Это предварительный шаг, который помогает инициализировать  основной алгоритм.
  2. **Configure mixture parameters (Сконфигурировать параметры смеси):**
     - **Описание:** Исследователь задает начальные параметры для модели смеси. Это может включать определение типов распределений для каждой компоненты (например, нормальное, экспоненциальное) и их начальных параметров.
     - **Отношение `include` к "Use initializer":** Этот use case включает в себя необязательный шаг использования инициализатора (Use initializer), который предоставляет начальное приближение параметров (например, на основе кластеризации).
  3. **Run algorithm (Запустить алгоритм):**
     - **Описание:** Основной use case, в рамках которого исследователь запускает процесс  оценки параметров смеси на основе сконфигурированной модели и данных.
     - **Отношение `extend` к "Configure algorithm":** Исследователь может детально настроить пайплайн алгоритма: выбрать семейство (EM-like, Monte Carlo), задать конкретные шаги (например, M-step на основе L-моментов), определить критерии останова и другие гиперпараметры.
  4. **Generate samples (Сгенерировать выборку):**
     - **Описание:** Вспомогательный use case, позволяющий исследователю сгенерировать синтетические данные из смеси с заранее известными параметрами. Это полезно для тестирования и валидации алгоритмов оценки.
